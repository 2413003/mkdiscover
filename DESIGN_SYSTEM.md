# MK Discover Design System

## Purpose

Define how every user-facing page must look and behave so the product stays simple, clear, and consistent.

This is a hard standard, not a suggestion.

## Core rule

A first-time user should understand each page in 1-2 seconds.

If they need to think hard, the design is wrong.

## Product feeling

Pages should feel:

- calm
- obvious
- premium
- fast
- trustworthy

## Page structure

Use this default structure for user-facing screens:

1. One short headline
2. Optional one-line support text
3. One primary action (search or CTA)
4. Results or content cards

Do not add extra sections unless they directly improve clarity.

## Non-negotiable UI rules

1. One primary action per screen.
2. Keep text minimal. Remove anything non-essential.
3. Prefer visual hierarchy over long explanation.
4. Keep navigation shallow and obvious.
5. Reuse consistent patterns across pages.
6. Design mobile-first, then scale to desktop.

## Visual rules

- Generous spacing and clear grouping
- Strong contrast and readable type
- Mostly neutral palette (black, white, gray)
- One accent color max
- Simple, consistent cards

Each result card should usually show only:

- title
- time (if relevant)
- location (if relevant)

## Interaction rules

- Instant feedback for key actions
- No unnecessary friction
- No login requirement for core discovery
- Obvious next action at all times

## Never allow

- Walls of text
- Feature-heavy dashboards for normal users
- Dense menus or hidden navigation logic
- Too many visible actions on one screen
- Technical language in user-facing copy
- Cluttered layouts

## Quality gate

Every page must pass all checks:

1. Can a first-time user instantly understand what this page is?
2. Can they see the next action immediately?
3. Can they act with minimal reading?
4. Does the page feel calm and uncluttered on mobile?
5. Does it match existing UI patterns?

If any answer is no, redesign before shipping.
